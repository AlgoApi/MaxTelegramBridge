</p>

**MaxTelegramBridge** — это высокопроизводительный асинхронный мост (UserBot), созданный для тех, кто ценит свой комфорт и свободу выбора. Если вас принуждают использовать мессенджер Max, этот инструмент позволит вам «эмигрировать» в привычный Telegram, сохранив формальное присутствие на навязанной платформе.

Проект зеркалирует сообщения, группы и каналы из Max в ваши приватные или публичные чаты Telegram, обеспечивая полную изоляцию от взаимодействия с Max.

### 🛠 ТЕКУЩИЙ ФУНКЦИОНАЛ (V1.0)
На данный момент проект работает в режиме **высокоскоростной ретрансляции**:
* **Личные сообщения:** Пересылка. 
* **Групповые чаты:** Пересылка.
* **Каналы:** Пересылка.
* **VIP-фильтрация:** Выбранные группы и каналы направляются в **другой канал Telegram**.
* **Полная поддержка медиа:** Фото, видео, голосовые сообщения и файлы.
* **QR-Авторизация:** Вход через сканирование кода (тип устройства: WEB).

### 🗺 РОУДМАП (ROADMAP)
Сейчас мост работает только на чтение, но это только начало:
- [ ] **RAM bloat:** Cтриминг в файл и отправку пути к файлу вместо загрузки в RAM.
- [ ] **Full-Reply Mode:** Ответ на сообщение в Telegram уйдет обратно в чат Max.
- [ ] **Button Sync:** Поддержка нажатия кнопок в сообщениях Max из Telegram.
- [ ] **Reaction Forwarding:** Синхронизация ваших реакций на посты.
- [ ] **Stealth Read:** Возможность читать сообщения в Telegram, не оставляя пометки «Прочитано» в Max.
- [ ] **Global Search:** Поиск по сообщениям Max через Telegram-бота.

### ⚙️ ЗАПУСК И НАСТРОЙКА (DOCKER)

#### 1. Переменные окружения (`.env`)
Настрой файлы для доступа к API:

```env
CURRENT_MAX_USERID=ваш id из Макса, пишется при запуске
MAX_PHONE=ваш номер телефона слитно без плюса
REDIS_HOST=
REDIS_PASSWORD=

# Telegram (регистрация на my.telegram.org)
TG_API_ID=1234567
TG_API_HASH=ваш_хеш

# Бот-ретранслятор (от @BotFather)
TG_BOT_TOKEN=ваш_токен

# Каналы назначения (ID через @getidsbot)
TG_CHANNEL_MAIN=-100123456789    # VIP-канала
TG_CHANNEL_SPECIFIC=-10098765432 # Общий-поток

# ID чатов из Max для общего-потока (через запятую) айди чата Max можно узнать из ссылки на чат в веб версии web.max.ru
SPECIFIC_MAX_GROUPS=id1,id2
SPECIFIC_MAX_CHANNELS=id3
```

#### 2. Быстрый старт через Docker
Скрипт полностью упакован для работы на Arch Linux или любом другом дистрибутиве с Docker:

```bash
# 1. Сборка образа
docker-compose build

# 2. Первый запуск (Интерактивный!)
# Обязательно без флага -d, чтобы отсканировать QR-код
docker-compose up
```

**Инструкция по QR:** В терминале появится QR-код. Откройте мобильный Max -> Настройки -> Устройства -> Связать новое устройство. Отсканируйте код. После сообщения об успехе остановите процесс (`Ctrl+C`).

```bash
# 3. Запуск в фоновом режиме
docker-compose up -d
```

#### 3. Мониторинг
Чтобы смотреть за работой моста в реальном времени:
```bash
docker logs -f max_tg_bridge
```
